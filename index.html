BUSINESS CASE

The eHotel Smart Reservation System provides a solution to the current inefficient reservation system. It maximizes financial profitability and customer satisfaction.

Profitability

Increased revenue would result from decreased costs as less employees would be needed since reservations would be done directly by the customer. Technology should automate low-level jobs, and employees would be performing monitoring and other value added activities. There would be less administrative work and more time for building relationships with customers. In essence. customer satisfaction and loyalty is key for business success.

In addition, management would be able see trends and patterns reservations. This would optimize the marketing strategy as they would make informed decisions regarding promotions, discounts and variable cost of the rooms.

Customer Satisfaction

The eHotel Smart Reservation System is flexible and accessible 24/7 for customers making reservations. There is no waiting time and it is simple to navigate online. Reduced waiting and time efficiency is linked to cost savings.

The centralized data storage ensures data correctness and validity, as well as providing backup archiving for analysis purposes and influencing marketing strategies. It prevents over/double bookings and manual data errors.

Notably, customer satisfaction leads to loyalty, acquisition and retention, which is vital to a business.

Efficiency

The eHotel Smart Reservation System has a user-friendly platform, is intuitive to navigate and easily manageable. It is interactive and displays real-time availability and pricing. Furthermore, security is ensured and all personal information stays private through data encryption processes.

REQUIREMENT SPECIFICATION

The eHotel chain has multiple hotels. Each hotel has rooms and employees. Rooms have to be reserved in advance. Rooms have variable costs. Customer data and reservation details are stored.

System Overview

The system is composed of the following modules:

  • Room Reservation Module – used by customers to make reservations
  • Operations Reporting Module – used by employees to gain detailed insight regarding room reservations to make operations
  • Management Reporting Module – used by managers to get reports from the data warehouse and to be able to make financial, marketing and strategy decisions
  • Predictive Analytics Module – usage of machine learning for predicting future hotel trends

Room Reservation Module

This module enables users to make reservations. The user selects the date, hotel, room type, length of stay. Date refers to the date of their estimated check-in date. There are different room types – including standard, double, family and suite.

Operations Reporting Module

This module provides employees visibility into the hotel room reservations. Reservation frequency and volume can be observed, relating to elements such as room type, hotel location. Dynamic factors such as time and room price are also displayed.

Management Reporting Module

This module provides data reports that show room reservations patterns. This is crucial for managers, as it influences their future business practices and decisions. Information is stored in the data warehouse regarding customer, hotel and room details.

Predictive Analysis Module

This module reveals hotel room reservations trends through analytics. Predictive models enhance business performance, customer retention and profitability. Room reservations data can be filtered and aggregated by date, hotel, location, room type.

DATABASE SCHEMA (SQL SERVER)

The following script was executed in SQL Server to create the database system:

CREATE TABLE Country
(
	Id INT PRIMARY KEY IDENTITY (1,1),
	Code NVARCHAR (10) NOT NULL,
	Name NVARCHAR (50) NOT NULL
);

CREATE TABLE Hotel
(
	Id INT PRIMARY KEY IDENTITY (1,1),
	Name NVARCHAR (50) NOT NULL,
	Address NVARCHAR (200) NOT NULL,
	PostCode NVARCHAR (10) NOT NULL,
	City NVARCHAR (50) NOT NULL,
	CountryId INT FOREIGN KEY REFERENCES Country(Id) NOT NULL,
	Phone NVARCHAR (10) NOT NULL	
);

CREATE TABLE Customer
(
	Id INT PRIMARY KEY IDENTITY (1,1),
	FirstName NVARCHAR (50) NOT NULL,
	LastName NVARCHAR (50) NOT NULL,
	Address NVARCHAR (200) NOT NULL,
	City NVARCHAR (50) NOT NULL,
	PostCode NVARCHAR (10) NOT NULL,
	DateOfBirth date NOT NULL,
	Phone NVARCHAR (10) NOT NULL,
	Email NVARCHAR (50) NOT NULL,
	IsMale bit NOT NULL
);

CREATE TABLE RoomType
(
	Id INT PRIMARY KEY IDENTITY (1,1),
	Name NVARCHAR (50) NOT NULL,
	Description NVARCHAR (250) NOT NULL,
	RoomPrice money NOT NULL,
	MaxGuests INT NOT NULL
);

CREATE TABLE Room
(
	Id INT PRIMARY KEY IDENTITY (1,1),
	RoomNumber INT NOT NULL,
	HotelId INT FOREIGN KEY REFERENCES Hotel(Id) NOT NULL,
	RoomTypeId INT FOREIGN KEY REFERENCES RoomType(Id) NOT NULL,--??
	IsSmoking bit NOT NULL
);

CREATE TABLE Reservation
(
	Id INT PRIMARY KEY IDENTITY (1,1),
	ReservationDate date NOT NULL,
	CustomerId INT FOREIGN KEY REFERENCES Customer(Id) NOT NULL,
	RoomId INT FOREIGN KEY REFERENCES Room(Id) NOT NULL,
	NumberGuests INT NOT NULL,
	CheckInDate date NOT NULL,
	CheckOutDate date NOT NULL
);

DATABASE DATA ANALYSIS & REPORTING (SQL SERVER)

Reservation Listing

The following query provides details regarding all reservations that have been created:

SELECT RR.ReservationDate, C.FirstName, C.LastName, R.RoomNumber, RR.NumberGuests, RR.CheckInDate, RR.CheckOutDate, H.Name HotelName
FROM Reservation RR
LEFT JOIN Customer C ON RR.CustomerId = C.Id
LEFT JOIN Room R ON RR.RoomId = R.Id
LEFT JOIN Hotel H ON R.HotelId = H.Id;

Reservation Listing for a specified time range

The following query provides details regarding all reservations that have been created in the time range between 02.05.2018 – 10.05.2018:

SELECT RR.ReservationDate, C.FirstName, C.LastName, R.RoomNumber, RR.NumberGuests, RR.CheckInDate, RR.CheckOutDate, H.Name HotelName
FROM Reservation RR
LEFT JOIN Customer C ON RR.CustomerId = C.Id
LEFT JOIN Room R ON RR.RoomId = R.Id
LEFT JOIN Hotel H ON R.HotelId = H.Id
WHERE RR.ReservationDate BETWEEN '2018-05-02' AND '2018-05-10'
ORDER BY RR.ReservationDate ASC;

Reservation Listing for a specific time range and specific hotel

The following query provides details regarding all reservations that have been created in the time range between 02.05.2018 – 10.05.2018 for Grand Hotel London:

SELECT RR.ReservationDate, C.FirstName, C.LastName, R.RoomNumber, RR.NumberGuests, RR.CheckInDate, RR.CheckOutDate, H.Name HotelName
FROM Reservation RR
LEFT JOIN Customer C ON RR.CustomerId = C.Id
LEFT JOIN Room R ON RR.RoomId = R.Id
LEFT JOIN Hotel H ON R.HotelId = H.Id
WHERE RR.ReservationDate BETWEEN '2018-05-02' AND '2018-05-10'
AND H.Id = 7

Total Count Hotels

Total Reservations per day for a specific hotel in a specific timerange

The following query provides details regarding total number of reservations that have been created in the time range between 02.05.2018 – 10.05.2018, on a daily basis, for Grand Hotel London:

SELECT RR.ReservationDate, COUNT(*) TotalReservations
FROM Reservation RR
LEFT JOIN Customer C ON RR.CustomerId = C.Id
LEFT JOIN Room R ON RR.RoomId = R.Id
LEFT JOIN Hotel H ON R.HotelId = H.Id
WHERE RR.ReservationDate BETWEEN '2018-05-02' AND '2018-05-10'
AND H.Id = 7
GROUP BY RR.ReservationDate
ORDER BY RR.ReservationDate ASC;

Number of reservations by Reservation Date, by Customer (from a business perspective, Daily Reservations by Customer)

The following query provides details from a business perspective, regarding Daily Reservations by Customer in the time range between 01.05.2018 – 20.5.2018, for Grand Hotel London:

SELECT RR.ReservationDate, C.FirstName, C.LastName, COUNT(*) TotalReservations
FROM Reservation RR
LEFT JOIN Customer C ON RR.CustomerId = C.Id
LEFT JOIN Room R ON RR.RoomId = R.Id
LEFT JOIN Hotel H ON R.HotelId = H.Id
WHERE RR.ReservationDate BETWEEN '2018-05-01' AND '2018-05-20'
AND H.Id = 7
GROUP BY RR.ReservationDate, C.FirstName, C.LastName 
ORDER BY RR.ReservationDate ASC, C.FirstName ASC, C.LastName ASC;

Number of reservations by Reservation Date, by Customer for a specific range for number of reservations

The following query provides details regarding the Customers who made more than one reservation and less than five reservations, in the time range between 01.05.2018 – 20.05.2018, for Grand Hotel London:

SELECT RR.ReservationDate, C.FirstName, C.LastName, COUNT(*) TotalReservations
FROM Reservation RR
LEFT JOIN Customer C ON RR.CustomerId = C.Id
LEFT JOIN Room R ON RR.RoomId = R.Id
LEFT JOIN Hotel H ON R.HotelId = H.Id
WHERE RR.ReservationDate BETWEEN '2018-05-01' AND '2018-05-20'
AND H.Id = 7
GROUP BY RR.ReservationDate, C.FirstName, C.LastName 
HAVING COUNT(*) > 1 AND COUNT(*) < 5
ORDER BY RR.ReservationDate ASC, C.FirstName ASC, C.LastName ASC;
